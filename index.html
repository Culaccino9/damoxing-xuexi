把知识点变成项目的方式

学习这类仓库时，不要只看 README。更有效的方法是把每章都转成一个小实验。

  1. 先读 README 确认章节目标和材料入口。
  2. 下载或打开对应章节教程，提取「输入、核心方法、输出」。
  3. 运行 Notebook 或脚本，记录环境依赖与报错。
  4. 把流程画成图：数据流、模型流、工具调用流、安全决策流。
  5. 最后形成自己的学习卡片：概念、代码、结果、风险。
